# AI活用ガイド草案: 資格学習PJでGitも一緒に学ぶ

## 目的

資格学習や自己研鑽をGitで管理することで、学習記録を残しながらGitの基本操作も身につける。

このPJでは、生成AIパスポートの学習計画を題材にして、Codex、Markdown、GitHubを組み合わせた学習管理を行う。

## なぜ資格学習にGitを使うのか

| 観点 | 効果 |
| --- | --- |
| 記録 | いつ何を学んだか履歴が残る |
| 振り返り | 学習計画や弱点の変化を追える |
| 共有 | 社内メンバーへ資料として共有しやすい |
| 安全性 | 変更前の状態に戻しやすい |
| 学習効果 | Gitの基本操作を実例で覚えられる |

## Gitで学べること

- ブランチを切って作業を分ける
- コミットで作業の区切りを記録する
- pushでGitHubに共有する
- Pull Requestで差分を確認する
- mainを安定版として扱う

## 初心者向けの考え方

| Git用語 | 学習PJでのイメージ |
| --- | --- |
| repository | 学習プロジェクトの保管場所 |
| branch | 作業用の別ルート |
| commit | 学習・資料更新の一区切り |
| push | GitHubへの保存・共有 |
| Pull Request | 変更内容の確認依頼 |
| merge | 安定版への反映 |

## おすすめ運用

1. `main` は安定版にする
2. 学習ログや資料更新は作業ブランチで行う
3. 変更をコミットする
4. GitHubへpushする
5. Pull Requestで差分を見る
6. 問題なければmainへ反映する

## Codexとの組み合わせ

Codexには以下を頼める。

- ブランチ名の提案
- コミットメッセージの提案
- 変更差分の要約
- Pull Request本文の作成
- Git操作手順の説明
- コンフリクト時の状況整理

## プロンプト例

```text
この変更内容に合うブランチ名とコミットメッセージを提案してください。
初心者にも分かるように、なぜその名前がよいかも説明してください。
```

```text
git status と git diff の結果をもとに、今回の変更内容を要約してください。
Pull Request本文として使える形にしてください。
```

```text
mainブランチに直接コミットしない運用にしたいです。
このプロジェクトに合うブランチ戦略と、初心者向けの手順書を作ってください。
```

## 注意点

- 機密情報や個人情報をリポジトリに入れない
- `.gitignore` を使って不要なファイルを管理対象外にする
- mainへ直接コミットしない練習をする
- `reset --hard` や `push --force` は安易に使わない
- 困ったらまず `git status` を見る

## 社内展開メッセージ

資格学習をGitで管理すると、学習の進捗が見えるだけでなく、Gitの基本操作も自然に練習できる。

Codexを使えば、Gitに慣れていない人でも、ブランチ名、コミットメッセージ、差分要約、手順確認を支援してもらえる。

AI活用とGit活用をセットで体験することで、学習管理、資料作成、ナレッジ共有の質を上げられる。
