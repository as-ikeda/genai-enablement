```text
ボルツマンマシン

```

## 生成モデルの誕生

### ボルツマンマシン

| 項目 | 内容 |
| --- | --- |
| 時期 | 1980年代 |
| 人物 | ジェフリー・ヒントン、テリー・セジュノスキーらの研究グループ |
| 概要 | 確率的なニューラルネットワークの一種 |
| ポイント | 生成モデルの初期の重要な流れとして押さえる |

メモ:

- ボルツマンマシンは処理に膨大な時間がかかるという課題があった。
- 1986年に制約付きボルツマンマシン（RBM）が開発された。
- RBMでは、ネットワークを入力部分と推定する部分に分けることで、学習が収束しやすくなった。
- 教師なし学習が可能になった点が重要。

要確認:

- 「0/1の2値を用いた教師あり学習から、実数を使った多層学習が可能になった」という理解は、公式テキストの表現に合わせて再確認する。

## 自己回帰モデルとディープラーニング

### 自己回帰モデル

| 項目 | 内容 |
| --- | --- |
| 概要 | 過去データを使って次のデータを予測するモデル |
| 得意分野 | 時系列データ |
| 例 | 株価、天気、文章の次の単語予測 |

自己回帰は「過去の流れから次を予測する」と理解する。

### ディープラーニング

| 項目 | 内容 |

| 概要 | 多層のニューラルネットワークを用いて複雑なパターンを学習する技術 |
| ポイント | 画像、音声、文章など、さまざまなモデルの基盤になった |

2006年には、制約付きボルツマンマシンを多段に重ねた深層ニューラルネットワークが提唱された。

## CNN（畳み込みニューラルネットワーク）

| 項目 | 内容 |
| --- | --- |
| 正式名称 | Convolutional Neural Network |
| 用途 | 画像認識、画像処理 |
| 特徴 | 画像の局所的な範囲を見ながら特徴を抽出する |
| 代表的な出来事 | 2012年、画像認識技術の世界大会で大きな成果 |

CNNは、通常のニューラルネットワークのようにすべての入力と出力を結びつけるのではなく、画像の一部分を見ることで効率よく特徴を抽出する。

| 層 | 認識する特徴のイメージ |
| --- | --- |
| 浅い層 | 線、色、輪郭 |
| 中間層 | 組み合わさった形状 |
| 深い層 | 物体、風景など複雑な特徴 |

覚え方:

```text
CNN = 画像に強い。局所特徴から全体特徴へ。
```

## VAE（変分自己符号化器）

| 項目 | 内容 |
| --- | --- |
| 正式名称 | Variational Autoencoder |
| 提唱 | 2013年、Diederik P. Kingma と Max Welling |
| 用途 | 次元削減、データ生成、異常検知 |
| 構成 | エンコーダとデコーダ |

### 構成

| 要素 | 役割 |
| --- | --- |
| エンコーダ | 入力データを潜在ベクトルに変換する |
| 潜在ベクトル | データの重要な特徴を表す低次元表現 |
| デコーダ | 潜在ベクトルから元データに近いものを生成する |

覚え方:

```text
VAE = 圧縮して、特徴を取り出して、もう一度生成する。
```

## GAN（敵対的生成ネットワーク）

| 項目 | 内容 |
| --- | --- |
| 正式名称 | Generative Adversarial Networks |
| 提唱 | 2014年、Ian Goodfellow |
| 用途 | 画像生成など |
| 構成 | 生成器と識別器 |

### 構成

| 要素 | 役割 |
| --- | --- |
| 生成器（Generator） | ランダムノイズや潜在空間のサンプルからデータを生成する |
| 識別器（Discriminator） | 生成データと本物データを見分ける |

GANは、生成器と識別器が競い合うことで、より本物らしいデータを生成できるようになる。

覚え方:

```text
GAN = 生成器 vs 識別器
```

要確認:

- 「大量データを必要としない」という理解は、試験向けには慎重に扱う。GANは教師なし寄りの生成モデルとして理解し、生成器と識別器の競争を優先して覚える。
- メモ中の「洗剤空間」は「潜在空間」。

## RNN（回帰型ニューラルネットワーク）

| 項目 | 内容 |
| --- | --- |
| 正式名称 | Recurrent Neural Network |
| 用途 | 時系列データ、文章、音楽など |
| 特徴 | 過去の情報を記憶しながら新しい入力を処理する |

RNNは、前の時刻の隠れ層の出力を、次の時刻の隠れ層の入力として利用する。

例:

- 文章
- 音楽
- 時系列データ
- 会話の流れ

課題:

| 課題 | 内容 |
| --- | --- |
| 勾配消失 | 長い系列では学習が進みにくくなる |
| 勾配爆発 | 学習が不安定になる |

この課題への改良としてLSTMが使われるようになった。

覚え方:

```text
RNN = 過去を覚えて次を処理する。長い系列は苦手。
```

補足:

- Pepperのような対話ロボットを考えると、過去の会話や状態を踏まえて応答するイメージとしては近い。ただし、Pepper全体をRNNそのものと考えるより、時系列・会話処理に使われる技術要素として理解する。

## LSTM（長短期記憶）

| 項目 | 内容 |
| --- | --- |
| 正式名称 | Long Short-Term Memory |
| 提唱 | 1997年、Sepp Hochreiter と Jürgen Schmidhuber |
| 用途 | 時系列データ、シーケンスデータ |
| 特徴 | 長期的な依存関係を学習しやすいRNNの改良版 |

LSTMはRNNの課題である、長い系列での学習のしにくさを改善するためのモデル。

課題:

- 長文になると精度が下がる場合がある
- 時系列で順番に処理するため、並列学習に向きにくい
- 大規模データの学習には不向きな面がある

この課題を背景に、Attentionを中心としたTransformerが登場した。

覚え方:

```text
LSTM = 長い文脈を覚えやすくしたRNN。ただし並列処理は苦手。
```

## Transformerモデル

| 項目 | 内容 |
| --- | --- |
| 登場 | 2017年 |
| 論文 | Attention Is All You Need |
| 研究者 | Ashish Vaswani ら |
| 影響 | 自然言語処理、LLMの発展に大きな影響 |
| 基本技術 | Self-Attention |

Transformerは、RNNやLSTMのように順番に処理するのではなく、入力全体を一度に見て重要な関係を捉える。

### 主な特徴

| 特徴 | 内容 |
| --- | --- |
| Attention Mechanism | 各単語が他の単語とどれくらい関係するかを重みづけする |
| 並列処理 | シーケンス全体を一度に処理でき、学習が高速化しやすい |
| 位置エンコーディング | 単語の順序情報をモデルに与える |
| エンコーダ・デコーダ | 入力を数値表現に変換し、出力を生成する |

覚え方:

```text
Transformer = Attentionで全体の関係を見る。並列処理できる。LLMの基盤。
```

## Transformer以降の派生モデル

### GPT

| 項目 | 内容 |
| --- | --- |
| 開発 | OpenAI |
| 初出 | 2018年6月 |
| 正式名称 | Generative Pre-trained Transformer |
| 特徴 | 大量のテキストを学習し、新しい文章を生成する |
| ベース | Transformer |

GPTは「文章を生成する」方向に強いモデルとして理解する。

### BERT

| 項目 | 内容 |
| --- | --- |
| 開発 | Google |
| 初出 | 2018年10月 |
| ベース | Transformer |
| 特徴 | 双方向に文脈を理解する |

BERTの特徴:

| 仕組み | 内容 |
| --- | --- |
| MLM | 文中の一部単語をマスクし、前後の文脈から予測する |
| NSP | ある文が別の文の直後に来るかを予測する |

覚え方:

```text
GPT = 生成に強い
BERT = 文脈理解に強い
```

| --- | --- |
| 開発 | Facebook AI |

| 特徴 | BERTをより多くのデータで訓練した改良モデル |

### ALBERT

| 項目 | 内容 |

| 項目 | 内容 |

### GPTの流れ

| 年 | モデル | ポイント |
| --- | --- | --- |
| 2015 | GPT研究開始 | 対話型AI・大規模言語モデルへの流れ |

### RLHF

| 項目 | 内容 |
| --- | --- |
| 正式名称 | Reinforcement Learning from Human Feedback |
| 意味 | 人間のフィードバックを使った強化学習 |
| 目的 | モデルの出力を、人間にとって望ましい方向へ調整する |

覚え方:

```text
RLHF = 人間の評価を使って、AIの回答をより望ましくする仕組み。
```

## 今日の理解

- 第2章は、生成AIの技術がどのように発展してきたかを追う章である。
- CNNは画像処理に強い。
- VAEはエンコーダとデコーダで、特徴を圧縮して生成する。
- GANは生成器と識別器を競わせる。
- RNNとLSTMは時系列・文章のような順序のあるデータに使われる。
- TransformerはAttentionを使い、並列処理しやすく、現在のLLMの基盤になった。
- GPTは生成、BERTは文脈理解に強いモデルとして区別する。
- ChatGPTでは、RLHFが対話品質の向上に関係する。
