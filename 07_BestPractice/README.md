# ベストプラクティス

生成AIを社内で安全かつ効果的に使うための観点を整理します。

## 基本方針

- 機密情報・個人情報・顧客情報を不用意に入力しない
- 生成結果は必ず人が確認する
- 根拠が必要な情報は一次情報を確認する
- 著作権、ライセンス、社内規程に配慮する
- プロンプトと結果を再利用できる形で残す

## チェックリスト

### 入力前

- [ ] 入力内容に機密情報が含まれていない
- [ ] 個人情報が含まれていない
- [ ] 顧客名・案件名などの扱いに問題がない
- [ ] 利用するAIツールの社内ルールに合っている

### 出力確認

- [ ] 事実関係を確認した
- [ ] コードや設定は実行・検証した
- [ ] 著作権やライセンスの懸念を確認した
- [ ] 社内向けに表現を調整した

## 社内展開で伝えること

- 生成AIは下書き・観点出し・レビュー補助として有効
- 判断責任は人間にある
- 速さだけでなく、品質と安全性を両立する

## Codexで学習計画をPJ化するときのコツ

- 目的、期限、使える時間、成果物を最初に伝える
- 資格取得だけでなく、実務活用や社内展開までゴールに含める
- 公式情報や教材URLを渡し、確認日を残す
- 学習時間は生活リズムに合わせて見直す
- 移動時間はインプット、業後と休日はアウトプット・問題演習に分ける
- AIが作った計画は週次レビューで調整する
- 成果物はMarkdownで残し、PowerPointや社内文書に転用しやすくする
