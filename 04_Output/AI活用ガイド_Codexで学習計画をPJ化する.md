# AI活用ガイド草案: Codexで資格取得・学習計画をPJ化する

## このガイドの目的

社内メンバーが生成AIをより身近に使えるように、Codexを使って資格取得や学習テーマをプロジェクト化する方法を整理する。

今回の生成AIパスポート受験計画を、自社での初回事例として扱う。

## 想定読者

- 資格取得や技術学習を計画的に進めたい人
- 学習内容をチームに共有したい人
- AIを使って情報整理や資料作成を効率化したい人
- Codexを業務外の自己研鑽にも活用してみたい人

## できること

| 活用場面 | Codexに頼めること |
| --- | --- |
| 目的整理 | なぜ学ぶのか、何を成果にするのかを整理する |
| 計画作成 | 試験日や期限から逆算して学習スケジュールを作る |
| 情報整理 | 公式情報、教材、参考資料を一覧化する |
| 進捗管理 | タスク、週次レビュー、弱点管理を作る |
| 成果物化 | 勉強会資料、活用ガイド、事例集に展開する |
| 再利用 | 他の資格・テーマ向けにテンプレート化する |

## 基本の進め方

1. 学習テーマ、目的、期限を伝える
2. 使える学習時間や制約を伝える
3. 教材や参考URLを渡す
4. フォルダ構成と管理ファイルを作ってもらう
5. 週次計画とタスクに落とす
6. 学習ログ、弱点、成果物を継続更新する
7. 社内共有できる形に整える

## 最初にCodexへ伝える情報

| 項目 | 例 |
| --- | --- |
| 目的 | 生成AIパスポートに合格し、社内展開資料も作りたい |
| 期限 | 2026-08-30に受験予定 |
| 使える時間 | 出社日の移動、業後、休日 |
| 教材 | 公式テキスト、公式シラバス、補助アプリ |
| 成果物 | 学習計画、学習ログ、社内向けAI活用ガイド |
| 制約 | 社内情報・個人情報は入力しない |

## プロンプト例

```text
資格取得に向けた学習計画を作りたいです。
目的は合格だけでなく、学習内容を社内勉強会や活用ガイドに転用することです。
受験日はYYYY-MM-DD、使える学習時間は週○時間です。
教材URLは以下です。
この内容をもとに、Gitリポジトリで管理しやすいMarkdown構成、週次スケジュール、タスク管理、社内展開用の成果物案を作成してください。
```

## 成果物テンプレート

| 成果物 | 保存先 | 内容 |
| --- | --- | --- |
| PROJECT_PLAN.md | 00_Project | 目的、ゴール、成果物、進め方 |
| TASKS.md | 00_Project | Doing、Next、Backlog、Done |
| WEEKLY_REVIEW.md | 00_Project | 週次の狙い、実績、次の行動 |
| EXAM_PLAN.md | 05_Exam | 試験概要、受験日、学習テーマ、弱点管理 |
| STUDY_SCHEDULE.md | 05_Exam | 週別学習計画、必要時間、累計チェック |
| PRACTICE_LOG.md | 03_Practice | 実務・学習でのAI活用事例 |
| OUTPUT_PLAN.md | 04_Output | 社内共有資料、勉強会、ガイドの計画 |

## 社内展開時のメッセージ

- AIは答えを出すだけでなく、計画を立て、情報を整理し、継続しやすい形にする補助として使える
- 個人の学習や調査も、記録方法を工夫すればチームのナレッジに変えられる
- 使い方のコツは、目的・期限・制約・成果物を具体的に伝えること
- 最終判断と社内共有前の確認は人が行う

## 注意事項

- 社内規程に反する情報を入力しない
- 顧客名、案件名、個人情報、機密情報は伏せる
- 試験情報や法務・セキュリティ情報は公式情報を確認する
- AIの出力はそのまま配布せず、社内文書として表現と正確性を確認する
